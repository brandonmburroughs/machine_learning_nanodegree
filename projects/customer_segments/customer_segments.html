Implementation: Outlier Detection¶

Detecting outliers in the data is extremely important in the data preprocessing step of any analysis. The presence of outliers can often skew results which take into consideration these data points. There are many "rules of thumb" for what constitutes an outlier in a dataset. Here, we will use Tukey's Method for identfying outliers: An outlier step is calculated as 1.5 times the interquartile range (IQR). A data point with a feature that is beyond an outlier step outside of the IQR for that feature is considered abnormal.

In the code block below, you will need to implement the following:

  • Assign the value of the 25th percentile for the given feature to Q1. Use np.percentile for this.
  • Assign the value of the 75th percentile for the given feature to Q3. Again, use np.percentile.
  • Assign the calculation of an outlier step for the given feature to step.
  • Optionally remove data points from the dataset by adding indices to the outliers list.

NOTE: If you choose to remove any outliers, ensure that the sample data does not contain any of these points!
Once you have performed this implementation, the dataset will be stored in the variable good_data.

Implementation: Dimensionality Reduction¶

When using principal component analysis, one of the main goals is to reduce the dimensionality of the data in effect, reducing the complexity of the problem. Dimensionality reduction comes at a cost: Fewer dimensions used implies less of the total variance in the data is being explained. Because of this, the cumulative explained variance ratio is extremely important for knowing how many dimensions are necessary for the problem. Answer:

  • The advantages of using a K-Means clustering algorithm is that it is fast to run, intuitive to understand, and straightforward to implement. It relies upon ideas that are intuitive to most (assigning points closest to a cluster to center to that cluster and updating the center after assignment). Additionally, it usually converges quickly.
  • The advantages of using a Gaussian Mixture Model clustering algorithm is that it provides a "soft" clustering algorithm and is more robust. This means that rather than all of one points "mass" being given to one cluster, the point is split between the clusters based on how likely (in the probability sense) it is that a point belongs to a cluster. So, a point close to the cluster center would be assigned a probability close to one of belonging to that cluster, while a point far from the cluster center would be assigned a probability close to zero of belonging to that cluster. This effectively creates a weighted average when updating the Gaussian mean and variance. This is more robust.
  • Given the previous observations about the wholesale customer data, we should use the Gaussian Mixture Model cluster algorithm because of the soft clustering it provides. This will allow us to disperse the effect of the remaining outliers and take advantage of the fact that many points in the dataset seem ambigious. From the visualizations we've created so far, it is not apparent that there are clear clusters in the data.
